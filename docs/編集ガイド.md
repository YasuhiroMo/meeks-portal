# 📖 メークスポータル コンテンツ編集ガイド

> このガイドは非エンジニアの方向けです。プログラミングの知識は不要です。

---

## 🎯 編集するファイルはこの4つだけ

| ファイル | 内容 | 場所 |
|---------|------|------|
| `manuals.js` | 手順書の全テキスト | `src/data/manuals.js` |
| `courses.js` | Eラーニングのコース・問題 | `src/data/courses.js` |
| `chatKnowledge.js` | AIチャットボットの知識 | `src/data/chatKnowledge.js` |
| `contacts.js` | 緊急連絡先・電話番号 | `src/data/contacts.js` |

**他のファイルは触らないでください。**

---

## ✏️ 編集の基本ルール

### 1. テキストの書き換え方
`""` （ダブルクォーテーション）または `` ` ` `` （バッククォート）の **中だけ** を書き換えます。

```javascript
// ✅ 正しい例
title: "新しいタイトル"

// ❌ ダメな例（" を消してしまった）
title: 新しいタイトル
```

### 2. 長いテキストの書き換え
`` ` ``（バッククォート）で囲まれた部分は改行OK：

```javascript
content: `ここに長いテキストを
改行しながら
自由に書けます`
```

### 3. 記号を消さないこと
`{}`  `[]`  `,`  `:` これらの記号は構造を維持するために必要です。消すとエラーになります。

## 📋 手順書を編集する（manuals.js）

### テキストを書き換える
```javascript
{
  title: "セクション名をここに",
  content: `
    手順の本文をここに書く。
    改行もそのまま反映される。
  `
}
```

### セクションを追加する
既存のセクションの後に `,` を付けて追加：
```javascript
sections: [
  { title: "既存セクション", content: `...` },
  { title: "新しいセクション", content: `ここに新しい内容` }  // ← 追加
]
```

## 🎓 クイズ問題を編集する（courses.js）

### 問題の構造
```javascript
{
  q: "問題文をここに書く",
  o: ["選択肢1", "選択肢2", "選択肢3", "選択肢4"],
  a: 0   // 正解の番号（0=選択肢1, 1=選択肢2, 2=選択肢3, 3=選択肢4）
}
```

### 例：正解が「選択肢3」の場合
```javascript
{
  q: "WBGT値がいくつ以上で外作業中止？",
  o: ["25", "28", "31", "35"],
  a: 2   // ← 0から数えて2番目 = "31"
}
```

### 問題を追加する
最後の問題の後に `,` を付けて追加：
```javascript
quiz: [
  { q: "既存の問題", o: ["A","B","C","D"], a: 0 },
  { q: "新しい問題", o: ["選択肢1","選択肢2","選択肢3","選択肢4"], a: 1 }  // ← 追加
]
```

## 🤖 チャットボットの知識を更新する（chatKnowledge.js）

`` ` `` で囲まれた長いテキストの中に、新しい情報を追記するだけ：

```javascript
export const KNOWLEDGE_BASE = `
...既存の内容...

【新しい項目】
ここに追加の知識を書く。
`;
```

## 🔄 編集後の反映手順

### 開発中（npm run dev 実行中）
→ **ファイルを保存するだけで自動反映** されます。

### 本番環境
```bash
npm run build     # ビルドを実行
```
→ `dist/` フォルダの中身を社内サーバーにコピー

## ⚠️ トラブルシューティング

| 症状 | 原因 | 対処 |
|-----|------|------|
| 画面が真っ白 | 構文エラー（記号の消し忘れ等） | 直前の編集を元に戻す |
| テキストが表示されない | `""` や `` ` ` `` が閉じていない | クォーテーションの対応を確認 |
| クイズの正解がおかしい | `a:` の番号間違い | 0から数えることを確認 |

**困ったらエンジニアに相談してください。**
